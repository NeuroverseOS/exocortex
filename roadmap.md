# Roadmap

3-month to 3-year view. Must align with the org roadmap.

## Now → 1 month

- **Symlinked layers** — Restructure exocortex into local/org/project layers. Org files live in a shared auki-context repo, symlinked into each personal exocortex. One file on disk, everyone reads it.
- **Project exocortices** — Every active Auki project gets a `.exocortex/` folder (readme.md, roadmap.md, sprint.md, glossary.md, changelog.md, src/). Contributors symlink projects into their personal exocortex.
- **Onboard the team** — Every project leader (BD, Marketing, DevRel, Spatial, Networking, Robotics) has a working exocortex with identity, role, values, and goals populated.
- **Cross-exocortex alignment** — CEO's exocortex can read team role files and project contexts. Structured audit process to review whether exocortices are making us more productive.
- **Auto-sync for org repo** — Git hooks or cron to keep auki-context current on every colleague's machine without manual pulls.

## 1 → 3 months

- **Exocortex via WhatsApp/Telegram** — AI partner reachable outside the IDE. Async nudges, attention reminders, daily check-ins.
- **Exocortex as standard infrastructure** — Every new hire sets up their exocortex on day one. It's part of onboarding, not an experiment.
- **Org-level CLAUDE.md** — Shared behavior rules that every agent inherits, with personal overrides on top.
- **Memetics book exocortex** — Structured external memory for the book on memetics, using the same local/org/project architecture.

## 3 months → 3 years

- Open-sourced exocortex framework adopted beyond Auki
- Exocortex-to-exocortex communication — agents that can query each other's project contexts with permission
- Exocortex as the standard onboarding and alignment layer for any organization using AI
